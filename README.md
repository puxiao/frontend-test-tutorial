**React 或 Vue ？**

本人平时使用 React，所以本系列文章以 React 项目测试为主，Vue 项目的测试不在本系列文章学习计划中。

* 针对 React 项目，我们将学习使用 react-testing-library
* 针对 Vue 项目，则推荐使用 Vitest



<br>

当然无论 React 还是 Vue，他们都是前端框架，所以绝大多数所用到的测试框架是相同、相通的。

例如我们要学习的针对 JS 测试的框架 Jest，Jest 同时支持对 React 和 Vue 。



<br>

我们先讲一些关于前端测试最基础的概念。



<br>

**为什么要学习前端测试？**

* 因为 Boss 直聘上很多前端招聘要求里都提到了需要会写单元测试
* 学习前端测试可以拉开自己与普通前端之间的差距
* 向开源项目提交代码时，需要读懂并通过他们的单元测试



<br>

**什么是前端测试？**

我个人将前端测试划分为 2 部分：

* 在开发阶段、项目编译之前，在 开发工具中(例如 VSCode) 中进行的 **单元测试**
* 在项目编译之后，在 浏览器 中进行的集成测试，也可以称呼为 **前端自动化测试**



<br>

**第1：单元测试 - 对功能单元进行的测试**

这里的功能单元也就是 "代码单元"，代码单元可以是一个函数、一个组件、一个网页等。

因此，我们可以笼统的将 单元测试 划分为：

* 函数 测试

  > 这里的 函数 包括 纯函数、有参数的函数、甚至是我们自定义的 hooks 函数

* 组件 测试

  > 这里的 组件 更加侧重的是 UI 组件

* 网络请求 测试

  > 准确来说还包括 为了测试 所要设计、准备的一些 虚假(mock)数据

* 数据状态 测试

  > 对于 react 来说，我个人推荐使用 zustand 来作为项目的 数据状态管理



<br>

**第2：集成测试(前端自动化测试) - 在浏览器中对编译后的前端页面进行测试**

集成测试、前端自动化测试、端对端测试，这些名词不同，但背后的含义却基本相同。

即：在 浏览器中 模拟真实用户的各种操作，尝试找出可能存在的问题(报错)

这里特别强调 3 点：

* "浏览器"：这里说的浏览器并不是指我们平时用的普通浏览器，而是指那些可提供调试接口、专门用作调试的浏览器，例如谷歌浏览器对应的 puppeteer 或 cypress
* "各种操作"：模拟真实用户的各种操作，这些操作包括 鼠标点击、输入框输入、提交表单、网络请求、页面切换等。
* "网络请求"：在测试时，我们没有必要真正请求后端接口，尤其是提交表单，没有必要把测试数据真的发送给后端，有专门模拟后端接口的第三方框架，例如 msw



<br>

除了单元测试、集成测试，在我的计划中也会将 前端监控 纳入学习计划中。



<br>

**仅限于前端调试？**

虽然我们讲的是前端调试，但其内核是 JS 代码调试，所以我们即将学习的各种调试技能完全适用于：

* node.js 后端项目
* electron 客户端项目



<br>

假定我们要针对 node 后端项目调试，那么我们需要针对不同的 node.js 版本、操作系统进行测试。

也就是意味着我们的测试电脑上需要安装不同的 node.js 版本。

目前 node.js 版本管理工具有：nvm、fnm(推荐) 等



<br>

> 尽管前端项目编译时也需要本机的 node.js 环境，但是我个人认为前端项目是没有必要去针对不同 node.js 版本进行测试的。
